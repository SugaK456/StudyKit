Goal & Scope — elementi obbligatori

  • Sistema prodotto e sue funzioni
  • Unità funzionale (FU) o Declared Unit (DU)
  • Confine del sistema (system boundary)
  • Procedure di allocazione
  • Categorie di impatto e metodologia LCIA
  • Requisiti qualità dati (10 dimensioni ISO)
  • Assunzioni, limitazioni, tipo di revisione critica

FU vs Declared Unit

Functional UnitPrestazione quantificata — funzione nota e definita
Declared UnitQuantità di prodotto — funzione non specificabile
Reference FlowQuantità concreta per realizzare la FU

Flussi elementari

Scambi tra tecnosfera e natura (ecosfera). Solo questi alimentano la LCIA.

INPUT dalla naturarisorse energetiche, minerali, acqua, uso suolo
OUTPUT verso la naturaemissioni in aria, acqua, suolo

Flussi di prodotto (economici)

Scambi tra processi nella tecnosfera. I rifiuti sono flussi di prodotto — restano in tecnosfera fino al trattamento.

Functional flowLo scopo del processo (prodotto outflow o rifiuto inflow)
Non-functional flowTutto il resto
Prodotto (valore ≥ 0)Qualcuno lo acquista o accetta gratuitamente
Rifiuto (valore < 0)Chi lo cede paga per farselo gestire

Tassonomia processi multifunzionali

Monofunctional productiong4 → 1 funzione
Coproductiong4 + g5 → 2 funzioni
Monofunctional wastew1 → 1 funzione
Combined wastew1 + w2 → 2 funzioni
Recyclingw1 + g3 → 2 funzioni
Combined waste + recyclingw1+w2+g2+g3 → 4 funzioni

g = good (bene, valore ≥ 0), w = waste (rifiuto, valore < 0)

Gerarchia ISO 14044 — 3 livelli

1°
Evitare l'allocazione
Suddivisione del processo in sub-processi monofunzionali oppure System Expansion. Arbitrarietà: nulla/minima.
2°
Partitioning fisico
Massa, volume, energia (LHV/HHV), moli. Riflette le relazioni fisiche reali.
3°
Partitioning economico
f_i = (p_i × m_i) / Σ(p_j × m_j). Volatile nel tempo. Ultima risorsa.

System Expansion

Allarga la FU per includere tutte le funzioni. Cambia la domanda di studio. Dati extra: nessuno.

Substitution

Sistema = in studio − sostituto. Mantiene la FU. Non è in ISO 14044 esplicitamente. Dati extra: sistema evitato.

Partitioning

Processi monofunzionali virtuali. Possono violare bilanci fisici. Dati extra: fattori di partizione.

Riciclo

Cut-off / Avoided burden / 50-50. Nessun approccio universalmente corretto.

Struttura procedurale LCIA

OBBLIGATORIO 1Selezione categorie, indicatori, modelli
OBBLIGATORIO 2Classificazione flussi → categorie (un flusso può contribuire a più categorie)
OBBLIGATORIO 3Caratterizzazione: Σ CF_i × m_i → profilo LCIA
OPZIONALENormalizzazione, raggruppamento, pesatura*

*Pesatura vietata in studi comparativi pubblici senza disclosure esplicita
